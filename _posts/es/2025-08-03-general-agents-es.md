[Manus](https://manus.im) se promociona como una herramienta de agente de IA general, pero probablemente no funcione tan bien.

Una razón es que es muy lento, realiza mucho trabajo innecesario y es ineficiente. Otra razón es que, si se encuentra con un problema complejo o toca un punto débil, es probable que falle en tu tarea.

Los agentes verticales funcionan muy bien porque están altamente especializados. Están diseñados para tareas muy específicas. Existen docenas de bases de datos y más de cien frameworks de desarrollo web como Spring. También hay numerosos frameworks web, como Vue o React.

[Dify](https://dify.ai) se centra en usar IA para conectar flujos de trabajo, empleando un método de arrastrar y conectar para definir flujos de trabajo de IA. Necesitan hacer mucho para conectar información, datos y plataformas.

Yo también he construido algunos agentes simples, como un agente de refactorización de código en Python, un agente corrector de gramática, un agente de corrección de errores y un agente de fusión de ensayos.

El código es muy flexible. Por lo tanto, Dify solo cubre una pequeña parte del espacio de ideas posibles.

Manus realiza tareas y muestra a los usuarios cómo funciona mediante un método VNC para mostrar una computadora.

Creo que el futuro se decantará por estos dos enfoques.

Con Manus, necesitas subir código o texto para realizar tareas, lo cual no es conveniente. Con Dify, debes construir flujos de trabajo usando arrastrar y soltar, similar a MIT Scratch.

¿Por qué Scratch no es tan popular como Python? Porque con Python puedes hacer muchas más cosas, mientras que Scratch está limitado a programas simples con fines educativos.

Dify probablemente tenga limitaciones similares.

Manus puede manejar muchas tareas simples. Sin embargo, para algunas tareas, especialmente aquellas que tocan sus puntos débiles, fallará.

Además, muchos programas o servicios requieren tiempo para configurarse. En el enfoque de Manus, este proceso es lento.

Como programador, uso IA con Python para construir mis agentes verticales. Este es el enfoque más simple para mí. También puedo configurar prompts y contextos para asegurar una salida relativamente estable de las APIs de LLM.

Manus y Dify también están construidos con estas APIs de LLM. Su ventaja es que ya tienen muchas herramientas o código listos para usar.

Si quiero construir un agente bot para Twitter, usar Dify podría ser más conveniente que construirlo yo mismo con tecnologías de código abierto.
