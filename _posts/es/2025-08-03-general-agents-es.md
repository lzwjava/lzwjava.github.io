[Manus](https://manus.im) se afirma que es una herramienta de agente de IA general, pero probablemente no funcione tan bien.

Una razón es que es muy lento, realiza muchas tareas innecesarias y es ineficiente. Otra razón es que si se encuentra con un problema complejo o golpea un punto débil, es probable que fracases en tu tarea.

Los agentes personalizados funcionan muy bien porque están altamente especializados. Están adaptados para tareas muy específicas. Hay docenas de bases de datos y más de cien marcos de desarrollo web como Spring. También hay numerosos marcos web, como Vue o React.

[Dify](https://dify.ai) se centra en utilizar la IA para conectar flujos de trabajo, empleando un método de arrastrar y conectar para definir flujos de trabajo de IA. Necesitan hacer mucho para conectar información, datos y plataformas.

También he construido algunos agentes simples, como un agente de refactorización de código Python, un agente de corrección de gramática, un agente de corrección de errores y un agente de fusión de ensayos.

El código es muy flexible. Por lo tanto, Dify solo cubre una pequeña parte del espacio de posibles ideas.

Manus realiza tareas y muestra a los usuarios cómo funciona utilizando un método VNC para mostrar una computadora.

Creo que el futuro se decidirá por estos dos enfoques.

Para Manus, necesitas subir código o texto para realizar tareas, lo cual no es conveniente. Con Dify, necesitas construir flujos de trabajo utilizando arrastrar y soltar, similar a MIT Scratch.

¿Por qué Scratch no es tan popular como Python? Porque con Python puedes hacer muchas cosas, mientras que Scratch está limitado a programas simples con fines educativos.

Dify probablemente tenga limitaciones similares.

Manus puede manejar muchas tareas simples. Sin embargo, para algunas tareas, especialmente aquellas que golpean las debilidades de Manus, fracasará.

Además, muchos programas o servicios tardan en configurarse. En el enfoque de Manus, este proceso es lento.

Como programador, utilizo la IA con Python para construir mis agentes personalizados. Este es el enfoque más simple para mí. También puedo configurar indicaciones y contextos para garantizar una salida relativamente estable de las APIs de LLM.

Manus y Dify también están construidos con estas APIs de LLM. Su ventaja es que ya tienen muchas herramientas o códigos listos para usar.

Si quiero construir un agente de bot de Twitter, usar Dify podría ser más conveniente que construirlo yo mismo con tecnologías de código abierto.
