[Manus](https://manus.im) est présenté comme un outil d'agent IA généraliste, mais il ne fonctionnera probablement pas très bien.

Une raison est qu'il est très lent, effectuant beaucoup de travail inutile et étant inefficace. Une autre raison est que s'il rencontre un problème complexe ou atteint une faiblesse, vous risquez d'échouer dans votre tâche.

Les agents verticaux fonctionnent très bien car ils sont hautement spécialisés. Ils sont conçus pour des tâches très spécifiques. Il existe des dizaines de bases de données et plus d'une centaine de frameworks de développement web comme Spring. Il y a aussi de nombreux frameworks web, tels que Vue ou React.

[Dify](https://dify.ai) se concentre sur l'utilisation de l'IA pour connecter des flux de travail, en employant une méthode de glisser-déposer pour définir les workflows IA. Ils doivent faire beaucoup pour connecter les informations, les données et les plateformes.

J'ai également construit quelques agents simples, comme un agent de refactorisation de code Python, un agent de correction grammaticale, un agent de correction de bugs et un agent de fusion de dissertations.

Le code est très flexible. Ainsi, Dify ne couvre qu'une petite partie de l'espace des idées possibles.

Manus exécute des tâches et montre aux utilisateurs comment il fonctionne en utilisant une méthode VNC pour afficher un ordinateur.

Je pense que l'avenir se stabilisera sur ces deux approches.

Avec Manus, vous devez téléverser du code ou du texte pour effectuer des tâches, ce qui n'est pas pratique. Avec Dify, vous devez construire des workflows en utilisant le glisser-déposer, similaire à MIT Scratch.

Pourquoi Scratch n'est-il pas aussi populaire que Python ? Parce qu'avec Python, vous pouvez faire tellement de choses, alors que Scratch est limité à des programmes simples à des fins éducatives.

Dify a probablement des limites similaires.

Manus peut gérer de nombreuses tâches simples. Cependant, pour certaines tâches, surtout celles qui touchent aux faiblesses de Manus, il échouera.

De plus, de nombreux programmes ou services prennent du temps à configurer. Dans l'approche de Manus, ce processus est lent.

En tant que programmeur, j'utilise l'IA avec Python pour construire mes agents verticaux. C'est l'approche la plus simple pour moi. Je peux également configurer des prompts et des contextes pour garantir une sortie relativement stable des API LLM.

Manus et Dify sont également construits avec ces API LLM. Leur avantage est qu'ils disposent déjà de nombreux outils ou codes prêts à l'emploi.

Si je veux construire un agent bot pour Twitter, utiliser Dify peut être plus pratique que de le construire moi-même avec des technologies open source.
