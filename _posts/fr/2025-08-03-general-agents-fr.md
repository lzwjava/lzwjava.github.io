[Manus](https://manus.im) se présente comme un outil d'agent général d'IA, mais il ne fonctionnera probabilmente pas très bien.

Une raison en est qu'il est très lent, effectuant beaucoup de travail inutile et manquant d'efficacité. Une autre raison est que s'il rencontre un problème complexe ou atteint un point faible, vous êtes susceptible d'échouer dans votre tâche.

Les agents verticaux fonctionnent très bien car ils sont hautement spécialisés. Ils sont conçus pour des tâches très spécifiques. Il existe des dizaines de bases de données et plus d'une centaine de frameworks de développement web tels que Spring. Il existe également de nombreux frameworks web, tels que Vue ou React.

[Dify](https://dify.ai) se concentre sur l'utilisation de l'IA pour connecter les flux de travail, en employant une méthode de drag-and-connect pour définir les flux de travail d'IA. Ils ont beaucoup à faire pour connecter les informations, les données et les plateformes.

J'ai également construit quelques agents simples, tels qu'un agent de refactorisation de code Python, un agent de correction de grammaire, un agent de correction de bogues et un agent de fusion de dissertation.

Le code est très flexible. Ainsi, Dify ne couvre qu'une petite partie de l'espace des idées possibles.

Manus effectue des tâches et montre aux utilisateurs comment il fonctionne en utilisant une méthode VNC pour afficher un ordinateur.

Je pense que l'avenir se stabilisera sur ces deux approches.

Avec Manus, vous devez télécharger du code ou du texte pour effectuer des tâches, ce qui n'est pas pratique. Avec Dify, vous devez construire des workflows à l'aide de glisserie et déposer, similaire à Scratch de MIT.

Pourquoi Scratch n'est-il pas aussi populaire que Python ? Parce que avec Python, vous pouvez faire tellement de choses, tandis que Scratch est limité à des programmes simples à des fins éducatives.

Dify a probablement des limitations similaires.

Manus peut gérer beaucoup de tâches simples. Cependant, pour certaines tâches, en particulier celles qui atteindre les faiblesses de Manus, il échouera.

De plus, de nombreux programmes ou services prennent du temps pour être configurés. Dans l'approche de Manus, ce processus est lent.

En tant que programmeur, j'utilise l'IA avec Python pour construire mes agents verticaux. C'est la méthode la plus simple pour moi. Je peux également configurer des invites et des contextes pour m'assurer d'une sortie relativement stable des API de LLM.
