[Manus](https://manus.im) est présenté comme un outil d'agent IA généraliste, mais il ne fonctionnera probablement pas très bien.

Une raison est qu'il est très lent, effectue beaucoup de travail inutile et est inefficace. Une autre raison est que s'il rencontre un problème complexe ou touche un point faible, vous êtes susceptible d'échouer dans votre tâche.

Les agents personnalisés fonctionnent bien car ils sont très spécialisés. Ils sont adaptés à des tâches très spécifiques. Il existe des dizaines de bases de données et plus de cent frameworks de développement web comme Spring. Il existe également de nombreux frameworks web, tels que Vue ou React.

[Dify](https://dify.ai) se concentre sur l'utilisation de l'IA pour connecter les flux de travail, en employant une méthode de glissière et de connexion pour définir les flux de travail de l'IA. Ils doivent faire beaucoup pour connecter les informations, les données et les plateformes.

J'ai également construit quelques agents simples, tels qu'un agent de refactorisation de code Python, un agent de correction grammaticale, un agent de correction de bugs et un agent de fusion d'essais.

Le code est très flexible. Ainsi, Dify ne couvre qu'une petite partie de l'espace des idées possibles.

Manus exécute des tâches et montre aux utilisateurs comment il fonctionne en utilisant une méthode VNC pour afficher un ordinateur.

Je pense que l'avenir se stabilisera sur ces deux approches.

Pour Manus, vous devez télécharger du code ou du texte pour effectuer des tâches, ce qui n'est pas pratique. Avec Dify, vous devez construire des flux de travail en utilisant le glisser-déposer, similaire à MIT Scratch.

Pourquoi Scratch n'est-il pas aussi populaire que Python ? Parce qu'avec Python, vous pouvez faire tant de choses, tandis que Scratch est limité aux programmes simples à des fins éducatives.

Dify a probablement des limitations similaires.

Manus peut gérer de nombreuses tâches simples. Cependant, pour certaines tâches, en particulier celles qui touchent aux faiblesses de Manus, il échouera.

De plus, de nombreux programmes ou services prennent du temps à configurer. Dans l'approche de Manus, ce processus est lent.

En tant que programmeur, j'utilise l'IA avec Python pour construire mes agents personnalisés. C'est l'approche la plus simple pour moi. Je peux également configurer des invites et des contextes pour garantir une sortie relativement stable des API LLM.

Manus et Dify sont également construits avec ces API LLM. Leur avantage est qu'ils ont déjà beaucoup d'outils ou de code prêts à l'emploi.

Si je veux construire un agent de bot Twitter, utiliser Dify pourrait être plus pratique que de le construire moi-même avec des technologies open source.
