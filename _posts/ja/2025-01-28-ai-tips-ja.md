* Cursor に例のコードをコピーし、コメントで文脈を提供する。

* 香港地域では、DeepSeek または Mistral サービスを使用することは許容されるが、ChatGPT、Claude、Gemini を使用することは許容されない。

* 地域の API ポリシーは、そのアプリのポリシーとほぼ同じである。

* Visual Studio Code の代わりに Cursor を使用する。

* 例えば、git マージ エディタのシナリオなど、Visual Studio Code を使用する必要がある場合もある。この場合、私は `git config --global core.editor "code --wait"` を使用する。

* Deepseek V3 のリリースから、AI ツールにサブスクライブする必要はない。

* Gemini または Grok を使用して、プロンプト「テキスト名を含む幸せな蛇の年を祝う画像を生成する」などで祝祭の祝賀画像を生成する。

* 時には、AI モデルにオリジナルのテキストを提供してテーブルを作成しても、出力の一部が入力と異なる場合がある。例えば、Cursor で Deepseek V3 モデルを使用して pip list のテーブルを生成すると、`1.极狐0` のようなバージョンが含まれることがある。ここでは、`极狐` は中国の GitLab プラットフォームを指す。

* Deepseek または Mistral API を使用して、プロンプト「あなたはプロの翻訳者です。Jekyll ブログ投稿のマークダウンファイルを英語から中国語に翻訳しています。{text}」などでタイトルを翻訳すると、誤った翻訳につながることがある。提供したテキスト以外にも、出力に過剰な翻訳が含まれることが多い。

* 時には Cursor の AI モデルが部分的に正しいテキストを提供することがあるが、それを受け入れることができる。AI モデルに追加の指示を与えると、正しい部分を再生成させることができる。

* 大規模言語モデルに過剰な文脈を提供しない方が良い。例えば、会話のダイアログ行を生成する場合、100 のポイントを提供しない方が良い。大規模言語モデルには、すでに膨大な量のデータが含まれている。

* 翻訳やダイアログの歌詞を生成するなどのタスクで十分な文脈を提供する場合、チェーン・オブ・ソート機能を使用しない方が良い。遅くなり、冗長で役に立たない回答になる可能性がある。

* チャットボットがユーザーの指示に従えるかどうかをテストする方法の一つは、英語で何かを説明させ、その後中国語で入力を続け、チャットボットが英語の出力を維持しているかどうかを観察することである。

* 大規模言語モデルに文脈を提供する代わりに、大規模なテキストやコードのデータセットに基づいてモデルを微調整し、微調整されたモデルを使用する。

* 1 週間、AI チャットボットを交互に使用して、その違いを理解する。1 つの AI ツールや最良のツールに固執しないで、より多くを学びたい場合。

* Grok、Gemini、ChatGPT、DeepSeek、Mistral、Perplexity、Claude を同時に開き、多くのタブを開き、彼らの回答を読み、質問をするのは楽しい。私たちは、回答を比較したり、興味に基づいてさらに質問をしたりすることができる。

* タスクが YouTube ビデオの要約である場合は、まず Gemini を使用する。X の情報が含まれている場合は、まず Grok を使用する。

* 本はますます役に立たなくなっている。AI チャットボットを使用して本を読む。彼らは、第1章または第2章、本の要約、または本の紹介を提供できる。

* 深い思考やチェーン・オブ・ソート（CoT）を使用して難しい問題を解決する代わりに、時には問題を簡略化して標準モデルが解決できるようにすることができる。

* AI とのチャットでは言語が重要であり、特に特定の地域やエリアが関係する場合は、関連する地域知識を使用するとより正確な情報を得ることができる。例えば、身分証明書の取得方法や車両の登録方法など。

* 最新情報、例えば新しくリリースされたライブラリ、コード、ドキュメントなどを扱う場合、複数の AI チャットボットで確認した方が良い。知識のカットオフと応答に微妙な違いがある可能性がある。

* エッセイを書いた後、AI チャットボットを使用して文法を修正し、Git を使用して差異を確認する。

* AI を強力に活用する方法の一つは、トピックについてのアイデアを書き出し、Grok に分析させ、包括的なガイドと提案を提供させることである。
