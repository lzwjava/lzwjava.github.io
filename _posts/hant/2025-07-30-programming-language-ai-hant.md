AI 現在如此強大，以至於我們需要從 AI 的角度重新思考一切。未來 10 年內，哪些程式語言會流行？

Python 絕對會。許多 AI 聊天機器人使用 Python 在瀏覽器中執行代碼，例如 Grok。Python 以其簡單性、易學性和不錯的效能而聞名，並被許多軟體專案採用。

Python 比 C++、Java 和 Rust 慢。Java 擁有龐大的社群。Rust 基於 C 開發。

我好奇是否有許多專案會被 Rust 重寫或取代。被重寫指的是參考舊專案並使用 Rust 實現相同功能。被取代則是指用 Rust 寫成的類似軟體取代了其他語言的軟體。

Rust 的語法相對複雜。但在 AI 時代，這不是什麼大問題，因為 AI 會幫助編寫代碼。對於複雜的語法，人類其實也沒有太多困難。

我認為印地語或泰米爾語相當複雜。但對於北印度人來說，印地語不是問題，而對於南印度人來說，泰米爾語也不是問題。

但對於像我這樣的中國公民來說，我認為學習這些語言是個大問題。

乍看之下，所有印地語字母在我看來都很相似。我認為印地語和阿拉伯語之間的差異，就像中文和日文之間的差異，或英語和西班牙語之間的差異。

程式語言之間的差異比自然語言之間的差異小。其中一個主要原因是程式語言僅在字母外觀上有所不同，而自然語言還包括發音。自然語言在兩個方面有所不同：字母集和發音。

程式語言只有約一個世紀的歷史，但自然語言有超過 100 個世紀。人們在某件事上花費的時間越多，差異就會越大。意見稍有不同的人會創造自己的版本。

這解釋了英語口音。在某些 TikTok 影片中，有人說最差的英語口音是伯明翰。

所以其實 Rust 沒有太大問題。它的效能相當好，因為它基於 C/C++。

效能對許多應用程式至關重要。如今，許多應用程式被數十億人使用。對於底層雲端運算基礎設施，它們的服務被調用無數次。因此，即使是微小的效能提升也能節省大量金錢。

Rust 有什麼缺點嗎？有人抱怨它很難學。學習曲線很陡峭。AI 帶來好消息，因為它幫助學習很多。

我不用知道太多關於 Rust 的知識。作為一名擁有 10 年經驗的軟體工程師，我可以使用 AI 來幫助編寫許多簡單的 Rust 應用程式。我只需要知道基本的 Rust 編譯指令，例如 `cargo` 和 `cargo build`。我甚至不需要了解太多 Rust 語法本身。

對於 Rust，可變性或借用模型並不會給我帶來麻煩。對於少於 200 行代碼的簡單應用程式，我可以直接向 AI 提供錯誤訊息，讓它修正錯誤。

但為什麼人們仍然大量使用 Python，如果 Rust 那麼好呢？因為 Python 在另一個方面很出色。它非常容易使用和學習。它擁有龐大的社群和許多庫。

Python 的效能仍然足夠好，可以支援數百萬甚至數千萬用戶的產品。大多數產品沒有那麼多用戶。如果你確實有那麼多用戶，你可以聘請 Rust 或 Java 程式設計師來優化效能。

Python 適合許多開發領域：機器學習、網頁開發、數學、教學和腳本。雖然 Python 不適合桌面應用程式，但 MicroPython 可用於樹莓派。

AI 時代的 Java 會怎樣？它也會不錯，因為它擁有龐大的用戶基礎和社群。AI 幫助很大。它被許多大公司使用。它們傾向於不更改主要程式語言。對於它們的一些大型舊專案，使用新程式語言重寫專案可能需要十年的努力。AI 會幫助這個過程，但過程仍然會很慢。

通常，大公司中的理性人士不會考慮更改主要程式語言。它們的主要業務在其他領域。它們不太關心技術。如果它們關心，它們會成為軟體或互聯網公司，並在開源社群中領先。然而，並非許多《財富》500強公司關心這一點。

由於 AI，將會有許多新創公司。新創公司喜歡做新事物，因此它們會嘗試新程式語言。在 AI 時代，敏捷程式語言將在中小企業中獲勝。

在演算法競賽中，最受歡迎的程式語言會改變嗎？C++ 已經主導這個領域數十年。在實際的演算法競賽中，你不能使用 AI。但我想在 AI 時代，會有更少的人參加。

由於這些人非常擅長編程，並且由於 AI 的緣故有許多機會，為什麼不更多的人建立實際產品供用戶使用，而不是練習演算法問題？即使是演算法競賽的 GOAT（Gennady Korotkevich），也選擇加入 Devin。

但演算法競賽可以是聰明程式設計師的放鬆或退休愛好。它就像象棋或籃球。人們做這些事是因為他們喜歡或需要，而不是為了其他原因。許多人在 30 歲或 40 歲時仍然打籃球。他們可能是為了健康或讓生活更有趣。

對於 iOS 和 Android，有 Java、Kotlin、Swift 和 Objective-C。由於 AI 的緣故，不會有重大變化，因為選擇有限。在終端用戶端，效能要求並不高。Google 和 Apple 對它們的平台有很高的控制權。如果 Google 和 Apple 不改變，程式設計師也不會改變。

但對於伺服器，有許多選擇。更適合 AI 的語言將獲勝。

程序式程式語言將比物件導向程式語言更受歡迎。程序式語言直接且容易被 AI 生成，而 OOP 語言有許多嵌套層級或設計模式。

AI 會讓程式語言更多嗎？我認為會。Zed 和 OCaml 將有更多用戶。LLMs/AI 非常擅長學習模式，因此很容易將專案重寫為其他語言。

未來，程式語言將面臨更多競爭。那些在效能、語法和編譯器品質方面表現良好的語言將天然變得更受歡迎。競爭就像 LLM 一樣。那些天然優秀的，例如 Claude 和 DeepSeek，會變得流行。

如果 AI 變得如此強大，以至於我們不再需要學習程式設計，那還需要很長時間。假設我們有一個非常大的專案，有 1,000 個 Java 檔案。AI 可能需要 10 年才能輕鬆完成這些任務。
