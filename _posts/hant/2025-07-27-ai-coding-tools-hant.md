最近，我成功地運行了 Claude Code，所以想分享我在這段旅程中選擇的工具。同時，我也收集了一些 [AI 工具技巧](ai-tips-en)。

我必須說我使用 Claude Code 的時間相對較晚。

[Claude 3.5 Sonnet](https://www.anthropic.com/news/claude-3-5-sonnet) 大約在 2025 年 2 月底推出。

我必須說，直到最近我才成功嘗試。其中一個原因是它需要使用 Anthropic API，而該 API 不支援中國的信用卡。

另一個原因是 [Claude Code Router](https://github.com/musistudio/claude-code-router) 已經推出。因此，這讓我最近的嘗試成功。

我一直聽到很多人說它有多好。我於 2025 年 7 月嘗試了 Gemini CLI。在與它進行幾次對話，要求它修正我的程式碼後，我再也沒有使用它。

我也嘗試過另一款軟體代理 Aider。我使用 Cursor 的時間不長，大約六個月後就不再使用，因為我發現 Cursor 的許多外掛不運作，因為它是基於 VSCode 建立的。我曾經嘗試過 Cline 一段時間，但並未廣泛使用。

我使用 Grok 3 beta 模型在 VSCode 外掛中，並搭配 Copilot 外掛使用自訂模型。感覺很棒。

我不認為 Claude Code 能改變我的習慣。但既然我能成功運行它，並且有耐心再嘗試幾次，就讓我們看看未來幾週我會有什麼感受。

我是一個挑剔的用戶，擁有十年的軟體工程經驗。因此，我希望工具在實際使用時能表現良好。我不在乎品牌，我只在乎它的實際日常實用性。
