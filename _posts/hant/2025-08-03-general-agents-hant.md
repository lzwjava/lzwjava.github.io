manus（https://manus.im）自稱是通用人工智慧代理工具，但表現可能不理想。

原因之一是它速度很慢，做了很多不必要的工作，效率低下。另一原因是如果遇到複雜問題或其弱點，您完成任務的可能性很低。

垂直代理在特定任務上表現很好，因為它們非常專門。例如，有幾十個資料庫和超過一百個 web 發展框架，如 Spring。也有許多 web 框架，如 Vue 或 React。

[Dify](https://dify.ai) 專注于使用人工智慧連接工作流程，使用拖放方法定義人工智慧工作流程。它們需要連接大量信息、數據和平台。

我 тоже建立過一些簡單的代理，如 Python 代碼重構代理、文法修正代理、錯誤修正代理和文本合併代理。

代碼非常靈活。因此，Dify 只覆蓋了可能想法空間的一小部分。

manus 通過使用 VNC 方法顯示電腦來執行任務和展示其工作方式。

我相信未來將 సంక్షేమ這兩種方法。

對于 Manus，您需要上傳代碼或文字才能執行任務，非常不方便。與之相比，Dify 需要使用拖放方式構建工作流程，類似于 MIT Scratch。

為什麽 Scratch 不如 Python estúdio？因為使用 Python，您可以做很多 things，而 Scratch acabou அதிகシティ质 simplle programs for educational purposes。

Dify 可能有類似的限制。

Manus 可以處理許多簡單的任務，但對于某些任務，especially those that hit Manus's weaknesses，它會失敗。

此外，許多程序或服務需要一段時間來設定。在 Manus 的方法中，這一過程很慢。

作為程序員，我使用 Python 和人工智慧構建自己的垂直代理。對于我來說，這是最簡單的方法。我還可以設定提示和內容，確保 llm APIs Output 相對穩定。
